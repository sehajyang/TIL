# 2. 생성자에 파라미터가 많다면 빌더를 고려하라.

Created: 2021년 2월 28일
Created by: Seha Jyang

- 파라미터 개수가 많아지면 클라이언트 코드를 작성하거나 읽기 어렵다.
- 자바 빈 패턴은 클래스를 불변으로 만들 수 없으며 get, set을 만들어줘야하기 때문에 좋지 않다.
- 빌더 패턴은 파이썬과 스칼라에 있는 named optional parameters 를 흉내낸 것
- 빌더 패턴은 계층적으로 설계된 클래스와 함께 쓰기에 좋다.
- **생성자나 static factory 가 처리해야 할 파라미터가 많다면 빌터패턴을 선택하는게 좋다**
